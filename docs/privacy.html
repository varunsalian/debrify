Privacy Policy

Privacy at Debrify

Debrify does not require a separate Debrify account. It connects to the providers and services you choose, stores the credentials needed to keep those integrations working on your device, and sends limited analytics to help improve the app.

1. No Debrify Account Required

Debrify does not require you to register a separate Debrify account to use the app. You may choose to connect third-party services such as Torbox, Real-Debrid, PikPak, WebDAV servers, Jackett or Prowlarr indexer managers, Trakt, Reddit, IPTV providers, or Stremio addons. Those integrations are optional and are only used when you enable them.

2. What Debrify Stores Locally on Your Device

Debrify stores app settings and integration data locally on your device using platform storage. Depending on which features you use, this may include:

  • Torbox API keys
  • Real-Debrid API keys and endpoint preference
  • PikPak email, password, access token, refresh token, device ID, captcha token, and user ID
  • WebDAV server URLs, usernames, and passwords
  • Jackett and Prowlarr server URLs, API keys, and indexer settings
  • Trakt access token, refresh token, username, and token expiry
  • Reddit access token, refresh token, and username
  • Playback progress, continue-watching state, playlists, favorites, downloaded file metadata, and app preferences
  • Configured Stremio addons, local Stremio TV catalogs, IPTV playlists, and remote-control pairing metadata

This information is stored locally so Debrify can access your chosen services, remember your settings, and restore your playback state. It is not used to create a central Debrify user profile.

3. What Debrify Sends to Third-Party Services

When you use an integration, Debrify sends the requests needed for that feature to the relevant service. For example:

  • Provider account requests to Torbox, Real-Debrid, or PikPak
  • WebDAV file listing and file streaming requests to your configured WebDAV servers
  • Indexer search requests to your configured Jackett or Prowlarr servers
  • Progress sync and list requests to Trakt
  • Video or subreddit requests to Reddit when Reddit integration is enabled
  • Catalog, metadata, and source requests to installed Stremio addons
  • Remote configuration requests to Debrify’s GitLab-hosted support configuration

The exact data sent depends on the feature you use. Those services process your requests under their own privacy policies and terms.

4. Analytics

Debrify sends privacy-focused usage analytics to Aptabase to help improve the app. Analytics events may include:

  • Event name
  • Random session ID
  • App version and build number
  • Operating system name and version
  • Device locale
  • Debug/release state
  • Limited event-specific properties related to app usage

Debrify does not intentionally send your provider API keys, refresh tokens, passwords, or similar credentials to Aptabase.

5. GitHub Pages Website Analytics

The Debrify website hosted on GitHub Pages uses GoatCounter for privacy-friendly website analytics. This helps measure page visits without adding a separate Debrify account system.

6. Data Sharing

Debrify does not sell your personal data. Debrify shares data only as needed to operate the integrations you choose and to run app/site analytics described above.

7. Your Choices

You can control your data by:

  • Not connecting optional integrations you do not want to use
  • Logging out of provider, WebDAV, indexer manager, Trakt, Reddit, or PikPak integrations
  • Removing stored keys or tokens inside the app settings
  • Clearing local app data from your device
